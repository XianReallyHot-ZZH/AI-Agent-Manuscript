AI Agent Technical Sharing

5 行循环，长成一个 Agent Gateway

基于 AI-Agent-ZeroToOne 与 OpenClaw-ZeroToOne：从最小 Agent Loop，到工具、记忆、可靠性、多 Agent 协作和生产级网关。

开场强调：这不是泛泛介绍 Agent，而是从两个教学项目的真实 Java 源码里抽出核心模式。核心问题：相同问题，不同工程取舍。

你每天用的 Claude Code，核心其实是一个循环

模型不是“被调用一次”的函数，而是在环境里反复观察、决定、行动。

  • 一次请求可能包含多轮 API 调用
  • 工具结果会回写给模型
  • 停止条件由 stop_reason 给出

大白话

Agent 就像一个会查资料、会敲命令、会修改文件的人。模型负责决定下一步，代码负责把“下一步”变成真实动作。

对不熟悉 Agent 的听众，先不要进入框架名词。先讲“循环 + 工具 + 结果回灌”这个最小闭环。

什么是 AI Agent？先别急着上框架

观察

读取用户输入、历史消息、文件、工具输出和环境状态。

决定

模型基于上下文选择：直接回答，还是调用工具。

行动

Harness 执行工具，并把结果作为下一轮输入送回模型。

核心抽象：模型在一个可行动环境里循环，而不是代码里塞满 if/else 智能。
“Harness”就是运行环境。它提供工具、权限、上下文、恢复、通信等能力，但不负责替模型思考。

消息投递安全：先落盘，再投递

OpenClaw S08：WAL 原子写入 vendors/OpenClaw-ZeroToOne/claw-4j/light-claw-4j/src/main/java/com/claw0/sessions/S08Delivery.java:175-195,240-255 
static void writeAtomically(Path target, String content) throws IOException {
    Path tmp = target.resolveSibling(".tmp." + ProcessHandle.current().pid()
            + "." + target.getFileName().toString());
    try {
        Files.writeString(tmp, content, StandardOpenOption.CREATE,
                StandardOpenOption.TRUNCATE_EXISTING);
        try (var fos = Files.newOutputStream(tmp, StandardOpenOption.SYNC)) {
            fos.flush();
        }
        Files.move(tmp, target, StandardCopyOption.ATOMIC_MOVE,
                StandardCopyOption.REPLACE_EXISTING);
    } finally {
        try { Files.deleteIfExists(tmp); } catch (IOException e) { /* ignore */ }
    }
}
本节 takeaway：生产级 Agent 的关键能力是“动作可治理，失败可恢复，消息不凭运气送达”。
WAL 的语义是 at-least-once：成功后 ack 删除文件，失败则保留并重试，耗尽后进 failed。

String truncateToolResult(String result, double maxFraction) {
    int maxChars = (int) (maxTokens * 4 * maxFraction);
    if (result.length() <= maxChars) return result;
    int cut = result.lastIndexOf('\n', maxChars);
    if (cut <= 0) cut = maxChars;
    String head = result.substring(0, cut);
    return head + "\n\n[... truncated (" + result.length()
            + " chars total, showing first " + head.length() + ") ...]";
}

Appendix A7

CommandQueue / LaneQueue 并发细节

enterprise LaneQueue：虚拟线程 worker vendors/OpenClaw-ZeroToOne/claw-4j/enterprise-claw-4j/src/main/java/com/openclaw/enterprise/concurrency/LaneQueue.java:149-175 
private void pump() {
    while (activeCount < maxConcurrency && !deque.isEmpty()) {
        QueuedItem item = deque.pollFirst();
        if (item.generation() != generation.get()) {
            item.future().cancel(false);
            continue;
        }
        activeCount++;
        int expectedGen = item.generation();
        Thread.ofVirtual().name("lane-" + name + "-worker").start(() -> {
            try { item.future().complete(item.task().call()); }
            catch (Throwable t) { item.future().completeExceptionally(t); }
            finally { taskDone(expectedGen); }
        });
    }
}
同一 lane 内 FIFO，不同 lane 之间互不阻塞。generation 用于 reset 后取消旧任务。
